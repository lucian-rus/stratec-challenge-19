Application functionality

At launch, the main class, Handler will create a GUI containing the controls of the application. The Import button will open a dialog that will allow the user to select the desired input file and get it's path. The Generate button will then create the input array by calling the GetInput class and will return it to the Handler class then further it to the Overlay class. By selecting any of the Solve buttons, the respective methods in the Overlay class are called resulting in an output that is sent back to the Handler class to be displayed on the console.

Algorithm description

The algorithm starts at position [1][1] of the array and checks for the existence of 1s. As soon as the first instance is found, the loops searching the array for entities are broken and the process starts by setting the starting position using the starting row as current row - 1 and the starting column as current column - 1 then by checking for the existence of 1s below the current position, saving it as current column in case it exists. The algorithm then proceeds to search for all the 1s on the current row, saving the maximum length of the row as it goes. When all the 1s are found, the algorithm goes back to the current column in the next row and starts looking for 1s to the left. If there are any, the algorithm moves the current position to the left and the starting column position at current column - 1. The maximum length of each row is saved (in order to get the maximum length of the entity) as well as the height of the entity, which will later be used to create the borders around and save its dimensions. Every 1 that is found is replaced by a 3 to avoid being detected again the next time the process starts. If either the maximum length or the height is 1, then the entity is declared to be noise, and will not be bordered nor its coordinates saved. After the whole process is finished, an entity is created and stored as an Algorithm class object in the Overlay class.

Every entity has multiple properties:

  • start_i value

    - represents the y coordinate of the entity

  • start_j value

    - represents the x coordinate of the entity

  • height value

    - represents the height of the entity

  • length value

    - represents the maximum of the entity

  • string

    - represents the string associated with the entity

  • string rotated by 90 degrees

    - represents the string associated with the entity rotated by 90 degrees

  • string rotated by 180 degrees

    - represents the string associated with the entity rotated by 180 degrees

  • string rotated by 270 degrees

    - represents the string associated with the entity rotated by 270 degrees

Notes

  • the maximum number of columns supported by the application is 1000 and the maximum number of rows is 500

  • the algorithm includes a flaw that was intentionally left in. instead of treating the case of going up as in (visualization 1) and making additional unnecessary steps, the problem can be solved while outlining the entity, simply replacing the 1s left out with 3s (visualization 2). as this step happens before creating the strings attached to the entity, the problem was easily bypassed
